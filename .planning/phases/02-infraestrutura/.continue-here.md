- [ ] CONSTRAINT: `.dch` format requires `@path` — `dict_to_inst()` needs both `@subtype` AND `@path` pointing to the GDScript file. Missing `@path` causes `CharacterResourceLoader` to crash on every scene load.
- [ ] CONSTRAINT: `SceneTransition Overlay` blocks all mouse input — A transparent `ColorRect` with default `mouse_filter=STOP` silently intercepts ALL mouse events in every scene after any transition. Fix: `mouse_filter = 2` (MOUSE_FILTER_IGNORE) on the Overlay node.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| `.dch` missing `@path` | Godot's `var_to_str(inst_to_dict(resource))` format requires both `@subtype` (class name) and `@path` (script path). Writing only `@subtype` causes `dict_to_inst()` to fail with "invalid instance dictionary format (missing @path)". | blocking | Always include both fields when writing `.dch` files manually. Template: `{"@subtype": "DialogicCharacter", "@path": "res://addons/dialogic/Resources/character.gd", ...}` |
| Transparent ColorRect blocking input | `ColorRect` with `mouse_filter=STOP` (default) blocks mouse events even at `alpha=0`. The SceneTransition overlay was on CanvasLayer 100, making ALL buttons on ALL scenes unclickable after any scene transition. | blocking | Any CanvasLayer overlay ColorRect used for fade effects MUST have `mouse_filter = 2` (MOUSE_FILTER_IGNORE) in its .tscn. |

<decisions_made>

- `.dch` file format uses Godot's `var_to_str(inst_to_dict())` — NOT JSON. Requires `@subtype` (class name) + `@path` (script path) + all @export variables. Color serialized as `Color(r, g, b, a)`.
- `.dtl` timeline format: plain text, one event per line. Text events: `CharacterName (portrait): text`. End event: `[end_timeline]`. Character identifier = filename without extension.

- CanvasLayer overlays used for fade effects MUST have `mouse_filter = MOUSE_FILTER_IGNORE`.
