         - Else (player, harmless): pass (no damage to player)

| Player ↔ Projectile spawn | Player spawns projectile at global_position + offset; offset isolation prevents self-collision |
| Projectile ↔ Enemy collision | Projectile triggers body_entered; checks body.is_in_group("enemies") before calling die() |
| Cooldown ↔ Input | _power_cooldown is float, decremented per frame; cooldown blocks use_power() while > 0 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-13 | **I** (Information Disclosure) | Projectile spawn position | accept | Spawn position is visible (intentional feedback); player sees projectile fire |
| T-04-14 | **T** (Tampering) | Cooldown bypass | mitigate | Cooldown is checked in use_power() before spawn; input rate naturally throttled to 2Hz (0.5s intervals) |
| T-04-15 | **T** (Tampering) | Projectile self-damage | mitigate | Collision layer isolation (player layer 1, projectile layer 2, enemy layer 3); projectile doesn't hit player layer |
| T-04-16 | **A** (Availability) | Audio file missing | mitigate | ResourceLoader.exists() check in play_sfx() prevents crash; logs warning instead |

**Rationale:** Sketch power is cosmetic/offensive gameplay mechanic with no economic value. Cooldown is purely cosmetic (soft throttle, not hard block). Collision isolation is hardware-level (Godot physics layer system).
