# Phase 4 — UI Design Contract: Mundo 2 — A Faculdade

> Visual and interaction contract for Godot 4 pixel art 2D game HUD, menus, and in-game UI elements.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Engine | Godot 4.4.x (GDScript) |
| Viewport | 320x180 pixels (16:9) |
| Rendering Mode | Nearest-neighbor (pixel-perfect) |
| Sprite Sizes | Player/NPCs: 32x32, Enemies/Collectibles: 16x16 |
| Snapping | 2D transforms snapped to pixel (no sub-pixel jitter) |
| Font System | Bitmap font (pixel art), TBD specific font name |
| Palette Approach | Handcrafted per-world (no automatic token generation) |

---

## Viewport & Screen Layout (320x180)

| Region | Element | Position | Size |

| Center-Top | Boss HUD (during boss) | (160, 20) centered | 200px width, bar 16px tall |

## Mundo 2 Visual Palette

| Element | Color | RGB / Hex | Usage |
|---------|-------|----------|-------|
| Dominant Background | Bege/Creme | #D4C5A9 (warm cream) | Campus/ateliê walls, sky |
| Secondary Surfaces | Light Beige | #E8DCC4 (lighter cream) | Floor tiles, platforms |
| Accent — Power Ready | Bright Orange | #FF9500 | Power icon when available |
| Accent — Power Cooldown | Gray | #808080 | Power icon during cooldown |
| Boss HUD — Critical | Deep Red | #CC2200 | Quality bar <70% |
| Boss HUD — Passing | Forest Green | #228B22 | Quality bar 70-99% |
| Boss HUD — Perfect | Gold | #FFD700 | Quality bar ≥100% |
| Text (Labels, Dialogue) | Dark Brown | #3D2817 | All text on light backgrounds |
| UI Border/Outline | Brown | #6B5D51 | Box outlines, separators |

**Palette Rationale:** Bege/creme reflects the architecture studio (ateliê) aesthetic. Caótico mas colorido expressed through scattered project sprites, not base colors. Bright orange and gray for power states match visual affordance (ready vs. waiting). Boss bar colors follow Mundo 1 progression.

## HUD Elements

### 1. Health Points Display (Top-Left, new for Mundo 2)

**Location:** (8, 8) on screen  
**Component:** Three 16x16 heart sprites in a row, 4px gap between each  
**Visual State:**
- Full heart: colored (e.g., red #FF4444)
- Empty heart: dimmed/outlined only (grayscale #808080)

**Logic:** Display updates when HP changes (3/3, 2/3, 1/3, 0/3 — triggers respawn)  
**Sprite Asset:** `assets/sprites/ui/heart_full.png` (16x16), `heart_empty.png` (16x16)  
**Script:** Attach to `CanvasLayer` node in each fase scene; wire to `Player.hp` signal

**Interaction:** No interaction — read-only display

---

### 2. Power Icon HUD (Top-Right, persistent during gameplay)

**Location:** (304, 8) — right edge of screen  
**Component:** 24x24 icon showing active power  
**Visual States:**
- Ready (cooldown = 0): full color icon, no dimming
- Cooldown: icon tinted gray (#808080, alpha 0.6) or ghost effect
- No power unlocked: empty square outline

**Display Text:** Below icon, 1-line label showing power name (e.g., "Sketch", "Amor")  
Font: bitmap 8px, dark brown text

**Icons per Power:**
| Power | Icon Design | Color (Ready) |
|-------|-------------|---------------|
| Amor | Rotating aura/heart shape | Bright Pink #FF1493 |
| Sketch | Folded paper sheet | Bright Orange #FF9500 |
| (Future powers) | TBD per phase | TBD per phase |

**Interaction:** 
- Z key activates power (if ready)
- Shift+Z cycles to next unlocked power (visual feedback: icon flashes 1-2 frames on cycle)
- No cooldown numeric display (only color state matters)

**Script:** Attach to `CanvasLayer` node; wire to `Player._current_power` and `Player._power_cooldown_timer`

---

### 3. Checkpoint Activation Indicator (Bottom-Left, transient)

**Location:** (8, 164) when visible  
**Component:** 16x16 caneca de café (coffee mug) sprite + floating label  
**Visual Effect:**
- Appears on screen for 2 seconds when checkpoint is activated
- Animation: blink (show/hide every 0.3s) or soft glow pulse (brighter/dimmer)
- Label: "+1 PV" in yellow #FFD700, positioned above mug, 1px outline for readability

**Sprite Asset:** `assets/sprites/ui/caneca_cafe.png` (16x16)  
**Text:** "Checkpoint reached!" or just icon + "+1 PV" above it  
**Fade-out:** After 2s, fade out over 0.5s (alpha decay)

**Interaction:** None — passive indicator

---

### 4. Boss HUD — Barra de Qualidade (Center-Top, boss phase only)

**Location:** (80, 20) — 160px wide, centered horizontally  
**Component:** Two bars stacked:
1. **Qualidade Bar** (top): colored progress bar showing current quality %
2. **Alvo Bar** (bottom, optional): thin line showing current target threshold

**Dimensions:** 
- Bar width: 200px
- Bar height: 12px (main quality bar), 4px (alvo line below)
- Font: bitmap 10px, dark brown

**Visual Design:**
- Background box: light beige (#E8DCC4) with dark border
- Label above: "Qualidade" (centered, 10px font)
- Percentage text: "75%" (right side of bar, 8px font)
- Alvo indicator: horizontal dotted line at current threshold (e.g., 70%, 85%)

**Color Progression (Quality %):**
- 0-69%: Red (#CC2200) — failing, must improve
- 70-99%: Green (#228B22) — passing range
- ≥100%: Gold (#FFD700) — perfect/overfulfilled

**Mechanics:**
- Starts at 0%, increases with each TFG item presented (+15-20%)
- Each correct dialogue choice: +10%
- Each wrong dialogue choice: -15%
- Alvo threshold rises dynamically during banca (Professor Perpétuo "adiciona requisito")
- Victory: ≥70% at end of banca
- Defeat: <70% at end of banca

**Script:** Attach to boss_tfg.gd; wire bar fill to quality percentage

**Interaction:** None — read-only during boss fight

## Typography

All text in Mundo 2 uses **bitmap pixel font** (8-10px sizes, no anti-aliasing).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| UI Label | 8px | Regular | 1.0 | HP count, power names, item labels |
| Dialogue | 8px | Regular | 1.2 | NPC dialogue, "Trouxe café..." lines |
| Boss Label | 10px | Bold | 1.0 | "Qualidade" header, percentage text |
| Cutscene Title | 12px | Bold | 1.2 | Opening narration, scene names |

**Font Asset:** `assets/fonts/pixel_font_8x8.tres` (Godot bitmap font resource)  
**Text Color:** Dark brown (#3D2817) on light backgrounds, white (#FFFFFF) on dark backgrounds  
**Outline:** 1px dark outline on all text for readability against varied backgrounds

## Accessibility Considerations

| Feature | Implementation |
|---------|-----------------|
| High Contrast | Dark brown text (#3D2817) on light beige (#E8DCC4) backgrounds meets WCAG AA |
| Text Size | 8px bitmap font readable at 320x180 viewport (tested by human playtest) |
| Color Blindness | Power states use color + icon shape (orange vs. gray icon distinctness) |
| Colorblind Modes | Deferred to Phase 12 (Polish) — COLOR_BLIND_MODE assist option |
