# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: `_came_from_pause` detection — Cannot use `get_tree().paused` in `options_menu._ready()` because `pause_menu.gd` calls `close()` (which sets `get_tree().paused = false`) **before** calling `SceneTransition.go_to(options_menu)`. By the time options_menu loads, the tree is already unpaused. The correct detection is via `SceneTransition.previous_scene` — check if the path contains "fase", "mundo", "boss", or "world".

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| ConfirmationDialog for in-game popups | OS-level dialogs do not respect Godot's 320×180 viewport scaling — they render at OS window size and clip/overflow | advisory | Always use `Panel` + `visible = true/false` for in-game confirmations; never `ConfirmationDialog.popup_centered()` |
| SaveManager key mismatch | `set_checkpoint()` saves to `"checkpoint_id"` but `_respawn()` was reading `"checkpoint"` — silent bug, no error, wrong behavior | advisory | Use constants for save keys, or search & replace rather than typing key strings manually |
| Signal connected but not to the right handler | `bike_choice.gd` emitted `chose_bicycle` and defined `_on_choice_made` but never connected them — game stayed paused permanently | advisory | After writing a signal emission and a handler, immediately verify `connect()` exists in `_ready()` |
