### T-06-03-04: Criar pause_menu.gd
**File:** `scenes/ui/pause_menu.gd`

```gdscript
extends CanvasLayer

@onready var _overlay: ColorRect = $Overlay
@onready var _panel: Panel = $Panel
@onready var _resume_btn: Button = $Panel/ButtonList/ResumeButton
@onready var _options_btn: Button = $Panel/ButtonList/OptionsButton
@onready var _quit_btn: Button = $Panel/ButtonList/QuitButton

var _open: bool = false


func _ready() -> void:
    _resume_btn.pressed.connect(close)
    _options_btn.pressed.connect(_on_options)
    _quit_btn.pressed.connect(_on_quit)


func _input(event: InputEvent) -> void:
    if event.is_action_pressed("pause"):
        get_viewport().set_input_as_handled()
        if _open:
            close()
        else:
            open()


func open() -> void:
    _open = true
    _overlay.visible = true
    _panel.visible = true
    get_tree().paused = true
    _resume_btn.grab_focus()


func close() -> void:
    _open = false
    _overlay.visible = false
    _panel.visible = false
    get_tree().paused = false


func _on_options() -> void:
    close()
    SceneTransition.go_to("res://scenes/options_menu/options_menu.tscn")


func _on_quit() -> void:
    close()
    AudioManager.stop_music()
    SceneTransition.go_to("res://scenes/main_menu/main_menu.tscn")
```

### T-06-03-07: Verificar conflito ESC com ControlsManager remap
O `options_menu.gd` já trata ESC para cancelar remap (`KEY_ESCAPE` no `_input`). O pause_menu também usa ESC via "pause" action.

Conflito potential: dentro do options_menu, ESC cancela remap E dispara o pause_menu.

Fix: Em `pause_menu.gd._input()`, verificar se estamos na cena `options_menu` antes de abrir/fechar:
```gdscript
func _input(event: InputEvent) -> void:
    if event.is_action_pressed("pause"):
        # Não abrir pause dentro de opções ou menu principal
        var scene_path := get_tree().current_scene.scene_file_path
        if "options_menu" in scene_path or "main_menu" in scene_path:
            return
        get_viewport().set_input_as_handled()
        if _open: close()
        else: open()
```

### T-06-03-08: Atualizar options_menu.gd — navegação de volta após pause
Quando o player abre opções VIA pause menu, o "VOLTAR" do options_menu deve retornar ao jogo (não ao main_menu).

Adicionar flag de origem:
```gdscript
# No options_menu.gd — detectar se veio do pause
var _came_from_pause: bool = false

func _ready() -> void:
    # Detectar se veio do pause verificando se há pause ativo
    _came_from_pause = get_tree().paused

func _on_back() -> void:
    if _came_from_pause:
        get_tree().back_to_scene()  # Godot 4: SceneTree não tem back_to_scene
        # Alternativa: SceneTransition.go_back() se existir, ou retornar para a cena salva
        get_tree().paused = false
        # Como não temos histórico de cenas, apenas fechar e retornar à cena anterior
        # Implementar SceneTransition.go_back() OU salvar cena anterior em SceneTransition
    else:
        SceneTransition.go_to("res://scenes/main_menu/main_menu.tscn")
```

**Alternativa mais simples:** No pause_menu, ao clicar OPCOES, em vez de navegar para options_menu.tscn, instanciar options_menu como overlay DENTRO do pause_menu. Mas isso duplica código.

**Solução recomendada:** Adicionar `SceneTransition.previous_scene` string para trackear cena anterior:
- Em `SceneTransition.go_to(path)`: salvar `previous_scene = get_tree().current_scene.scene_file_path` antes de transitar
- Em `options_menu._on_back()`: `SceneTransition.go_to(SceneTransition.previous_scene)` se veio do jogo

Verificar se `autoloads/scene_transition.gd` tem suporte a isso ou adicionar.
